Asset Management System Overview

The Knowledge Engine Asset Management System provides comprehensive tracking and management of datacenter infrastructure. Built with Django and designed for operational excellence, it integrates seamlessly with the existing user management system to provide role-based access control and audit trails.

Core Capabilities

  • Asset Inventory Management – Complete lifecycle tracking from procurement to decommission
  • Maintenance Scheduling – Preventive and corrective maintenance workflows
  • Technical Specifications – Flexible specification storage for different asset types
  • Status Monitoring – Real-time status tracking with visual indicators
  • Audit Trails – Complete change history and activity logging
  • User Integration – Asset assignment and responsibility tracking

Supported Asset Types

The system is specifically designed for datacenter infrastructure management, supporting:

Power Systems

  • • Generators (Diesel, Gas)
  • • Power Distribution Units

AssetSpecification Model

Flexible storage for technical specifications that vary by asset type:

class AssetSpecification(models.Model):
    asset = models.ForeignKey(Asset, related_name='specifications')
    specification_name = models.CharField(max_length=100)
    specification_value = models.CharField(max_length=255)
    unit = models.CharField(max_length=50, blank=True)
    
    class Meta:
        unique_together = ['asset', 'specification_name']

Operational Workflows

The Asset Management System supports key datacenter operational workflows, integrating with ITIL best practices and organizational procedures.

Asset Lifecycle Management

1. Asset Procurement & Registration

  1. Create new asset record with basic information
  2. Assign unique asset tag following naming convention
  3. Record manufacturer details and warranty information
  4. Add technical specifications and documentation
  5. Assign to responsible team member

2. Installation & Commissioning

  1. Update asset status to "Under Installation"
  2. Record installation date and location
  3. Perform commissioning tests and validation
  4. Update status to "Active" upon successful commissioning
  5. Schedule initial preventive maintenance

3. Operational Maintenance

  1. System automatically flags maintenance due dates
  2. Maintenance team schedules work orders
  3. Asset status updated to "Under Maintenance"
  4. Work performed and parts used are documented
  5. Next maintenance date scheduled upon completion

Maintenance Workflows

Comprehensive maintenance management supporting different maintenance types:

Preventive Maintenance

Preventive Maintenance Workflow:
1. System identifies assets due for maintenance
2. Maintenance planner schedules work order
3. Technician receives assignment notification
4. Asset status updated to "Under Maintenance"
5. Work performed according to procedures
6. Documentation completed with parts/costs
7. Asset returned to "Active" status
8. Next maintenance date calculated and scheduled

Corrective Maintenance

Corrective Maintenance Workflow:
1. Issue identified through monitoring or incident
2. Asset status updated to "Faulty"
3. Emergency work order created
4. Technician dispatched for diagnosis
5. Repair work performed with documentation
6. Testing and validation completed
7. Asset returned to operational status
8. Root cause analysis and prevention measures
